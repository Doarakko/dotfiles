# セッション意思決定記録コマンド

現在のセッションで行った技術的意思決定を振り返り、構造化されたマークダウンファイルに記録する。

## 使用方法
```
/save-session-decisions
/save-session-decisions docs/decisions
```

## 出力先
- 引数が指定された場合: `$ARGUMENTS` ディレクトリ配下に保存
- 引数なし: `docs/decisions/` ディレクトリ配下に保存
- ファイル名: `session-decision-YYYY-MM-DD-<セッションの主要トピックを英語で要約>.md`

## 手順

### 1. セッションの振り返り
現在のセッションの会話全体を振り返り、以下の技術的意思決定を抽出する:

- **技術選定**: ライブラリ、フレームワーク、ツールの選択とその理由
- **設計判断**: アーキテクチャ、デザインパターン、データ構造の選択
- **実装方針**: 実装アプローチ、アルゴリズムの選択
- **トレードオフ**: 検討した代替案と、採用案を選んだ理由
- **コーディング規約**: 命名規則、コードスタイルに関する決定
- **見送った案**: 不採用とした案とその理由

### 2. コード変更の確認
セッション中にコード変更があった場合、変更内容からも意思決定を補完する:
- `git diff` と `git diff --cached` で現在の変更を確認
- `git log --oneline -20` で直近のコミットを確認

### 3. 出力ディレクトリの準備
- 出力先ディレクトリが存在しない場合は `mkdir -p` で作成する

### 4. ファイルの作成
以下のフォーマットでマークダウンファイルを作成する:

```markdown
# 技術的意思決定記録

- **日付**: YYYY-MM-DD
- **セッション概要**: （このセッションで行った作業の1-2行の要約）

## 意思決定一覧

### 1. （意思決定のタイトル）

- **カテゴリ**: 技術選定 | 設計判断 | 実装方針 | コーディング規約
- **ステータス**: 決定済み | 検討中
- **決定内容**: （何を決定したかの簡潔な説明）
- **背景・課題**: （なぜこの決定が必要だったか）
- **検討した選択肢**:
  1. （選択肢A）: メリット / デメリット
  2. （選択肢B）: メリット / デメリット
- **決定理由**: （この選択肢を採用した理由）
- **影響範囲**: （この決定が影響するファイル・コンポーネント）

### 2. ...

## 未解決事項
- （セッション中に結論が出なかった技術的課題があれば記載）

## 関連リソース
- （参照したドキュメント、Issue、PRなどがあれば記載）
```

### 5. 結果の表示
- 作成したファイルのパスを表示
- 記録した意思決定の数を表示
- 内容のサマリーを表示

## ルール
- 意思決定が見つからない場合は、その旨をユーザーに伝え、ファイルは作成しない
- 事実に基づいて記録し、セッション中に議論されていない内容は含めない
- 「検討した選択肢」は、セッション中に実際に言及された選択肢のみ記載する
- 日本語で記載する（ファイル名のトピック部分のみ英語）
